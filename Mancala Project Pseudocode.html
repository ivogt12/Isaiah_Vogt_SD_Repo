Minimum Valuable Product:

AAU: I want to see a message that tells me whether I win, lose, or tie
AAU: I want to see a message that asks me if I want to play again
AAU: I want to see who's turn it is

ICEBOX:
AAU: I want the marbles in each default pocket(at the start of the game) to be one of four assorted colors
AAU: I want a timer to see hour long the game lasts
AAU: I want there to be a sound effect when the marble moves

PSEUDOCODE:
AAU: I want to click a pocket and have the marbles displace one-by-one down the board properly
    -will need an element to represent the board
    -will need an event listener attached to the board element
    -will need an element to represent all 48 marbles
    - will need to check if the space is available to click
    -will need some kind of turn state
    -will need a change turn function
    -will need to update the board state to reflect a click that took place
    -will need an animation that moves a marble element to the next pocket
    -will need to a winner state
    -If there is a winner, we would update a winner state variable and stop the game
    -If there no winner, we would change the turn state
    -we would then call the render method to update our DOM to reflect the new state

AAU: I want to see who's turn it is
    -text that changes based on the turn state to display the current turn

AAU: I want to see a message that tells me whether I win, lose, or tie
    -the winner state must be changed to the active turn if the checkWinner() function detects a winner based on the amount of marbles left
GAME END
    -Render win message dependent on updated winner state variable
    -Render tie message if both players have the same amount of marbles at the end

AAU: I want to see a message that asks me if I want to play again
    -all of the marble elements being gone from the board would trigger the play again button
    -Play again would be a clickable button
    -the button would have an eventListener that would trigger an init function 
    -The init function would clear the board state, reset the turns, reset the winner state, and render the board state to the DOM

UI:
    -the board and marbles must always be present on the screen
